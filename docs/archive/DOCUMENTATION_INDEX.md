## 📝 文檔維護原則

### 更新頻率
- **核心文件**（根目錄）：經常更新，保持最新
- **設置指南**（docs/setup/）：穩定，偶爾更新
- **ML 引擎**（docs/ml_engine/）：隨開發更新
- **Discord**（docs/discord/）：隨服務更新
- **測試文檔**（docs/testing/）：測試後更新
- **架構文檔**（docs/architecture/）：架構變更時更新
- **歸檔文件**（docs/archive/）：只讀，不再更新

### 新增文檔規則
根據文檔類型放置到對應目錄：
- 臨時筆記 → `docs/archive/sessions/`
- 狀態報告 → 更新 `PROJECT_STATUS.md`（不新增檔案）
- 修復紀錄 → `docs/archive/fixes/`
- 測試報告 → `docs/testing/`
- ML 相關 → `docs/ml_engine/`
- Discord 相關 → `docs/discord/`
- 設置相關 → `docs/setup/`
- 架構設計 → `docs/architecture/`

### 文檔清理
定期檢查並歸檔：
- 過時的狀態報告
- 已完成的 TODO
- 臨時會議記錄
- 已解決的問題報告
